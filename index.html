Why these mappings?
Use SAML for web SSO between an IdP (e.g., Azure AD) and enterprise SaaS SPs. Use OpenID Connect when you need authentication for apps (an identity layer on OAuth 2.0), typically via Authorization Code with PKCE. Use OAuth 2.0 to delegate API access (scopes), e.g., server-to-server (Client Credentials) or user-consented access (Auth Code + PKCE). Kerberos provides ticket-based auth in on‑prem AD. RADIUS/EAP-TLS is common for 802.1X network access control. Device Code enables sign-in on devices without an easy browser. Avoid Implicit and ROPC except legacy edge cases.
